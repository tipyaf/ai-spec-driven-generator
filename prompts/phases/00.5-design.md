# Phase 0.5: UX/UI Design

## Responsible agent
`ux-ui`

## Objective
Design the user experience and specify the visual interface before development.

## Prerequisites
- Phase 0 (Scoping) validated
- YAML spec with features and acceptance criteria

## Instructions

You are in **Phase 0.5 — Design**. The spec is ready. Your role is to design the UX and UI specs.

### Step 1: Information architecture
1. Read the spec and identify all necessary pages/screens
2. Create the sitemap (page hierarchy)
3. Define the main navigation

### Step 2: User flows
1. For each must-have feature, draw the user journey
2. Identify all states: empty, loading, error, success
3. Document transitions between pages

### Step 3: Design system
1. Define the color palette
2. Define typography
3. Define spacing and border radius
4. Choose a visual style consistent with the project

### Step 4: Component specification
1. For each UI component identified in the spec:
   - Textual wireframe (ASCII art)
   - Props and data
   - All states (default, hover, loading, error, empty)
   - Responsive behavior
   - Accessibility

### Step 5: Page layouts
1. For each page: layout with component placement
2. Mobile and desktop versions

### Step 6: Validate
1. Present the complete design
2. Request validation

## Expected deliverable
- Design document (sitemap, flows, design system, component specs)
- Saved in `output/[project-name]/docs/design.md`

## Validation criteria
- [ ] All necessary pages are identified
- [ ] Each component has its states defined
- [ ] Design is responsive (mobile-first)
- [ ] Accessibility is accounted for
- [ ] User has validated
