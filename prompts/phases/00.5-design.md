# Phase 0.5: UX/UI Design

## Responsible agent
`ux-ui`

## Objective
Design the user experience and specify the visual interface before development.

> **Platform scope**: This phase applies to UI projects (web, mobile, desktop). For non-UI projects (API, CLI, library, embedded, data pipeline), adapt this phase to define the API surface / command structure / module architecture / state machine design instead. Skip visual-specific steps (wireframes, design system, WCAG) and focus on interface contracts and interaction patterns.

## Prerequisites
- Phase 0 (Scoping) validated
- YAML spec with features and acceptance criteria

## Instructions

You are in **Phase 0.5 — Design**. The spec is ready. Your role is to design the UX and UI specs.

### Step 1: Information architecture
1. Read the spec and identify all necessary interfaces (web: pages, mobile: screens, CLI: commands, API: endpoints, library: modules)
2. Create the information architecture (web: sitemap, mobile: screen map, CLI: command tree, desktop: menu structure, API: endpoint map)
3. Define the main navigation or interaction entry points

### Step 2: User flows
1. For each must-have feature, draw the user journey
2. Identify all states: empty, loading, error, success
3. Document transitions between pages

### Step 3: Design system
1. Define the color palette — **include contrast ratio and WCAG AA status for every color pair**
2. Define typography
3. Define spacing and border radius
4. Choose a visual style consistent with the project

### Step 4: Component specification
1. For each UI component identified in the spec:
   - Textual wireframe (ASCII art)
   - Props and data
   - All states (default, hover, loading, error, empty)
   - Responsive behavior
   - Accessibility

### Step 5: Page layouts
1. For each page: layout with component placement
2. Mobile and desktop versions

### Step 6: Validate
1. Present the complete design
2. Request validation

## Expected deliverable
- Design document (sitemap, flows, design system, component specs)
- Saved in `specs/[project-name]-ux.md`

## Validation criteria
- [ ] All necessary pages are identified
- [ ] Each component has its states defined
- [ ] Design is responsive (mobile-first) — **web/mobile UI projects only**
- [ ] Accessibility is accounted for (ARIA roles, labels, focus management) — **web/mobile UI projects only**
- [ ] **WCAG 2.1 AA contrast ratios verified** for all color pairs — **web/mobile UI projects only**:
  - Normal text (< 18pt): ≥ 4.5 : 1
  - Large text (≥ 18pt) and UI components: ≥ 3 : 1
  - Focus indicators: ≥ 3 : 1
- [ ] Color table includes contrast ratio column with pass/fail status — **UI projects only**
- [ ] User has validated
