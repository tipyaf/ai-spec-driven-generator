# Phase 3.5: Validate (Auto-Validated)

## Responsible agent
`validator`

## Objective
Independently verify that the developer's implementation matches the spec. This phase is MANDATORY and fully automated — the developer NEVER validates their own work.

## Prerequisites
- Phase 3 (Implement) completed — developer declares task done
- Implementation manifest from Phase 1 (files, pages, endpoints to verify)
- Acceptance tests from spec

## Instructions

You are in **Phase 3.5 — Validate**. The developer has completed implementation. Your role is to independently verify everything works.

### Step 1: Gather context
1. Read the spec's acceptance criteria and acceptance_tests
2. Read the git diff to identify all changed files
3. Read the architect's implementation manifest

### Step 2: Visual checks (UI projects only)
1. Start dev server if not running
2. Screenshot every modified page/screen
3. Verify design system compliance (no hardcoded colors, correct tokens)
4. Check responsive breakpoints if web project
5. Verify WCAG contrast ratios if UI project

### Step 3: Code checks (all projects)
1. Grep modified files for anti-patterns from manifest
2. Verify no debug artifacts (console.log, debugger, TODO)
3. Check i18n compliance (no hardcoded user-facing strings)
4. Verify imports are used, no dead code

### Step 4: Runtime checks (API/web/CLI projects)
1. For API: curl endpoints, verify response status and shape
2. For web: verify pages load without errors
3. For CLI: run commands, verify output
4. For library: run example usage

### Step 5: Acceptance tests
Execute each acceptance_test from the spec:
- visual → screenshot + compare
- runtime → curl/invoke + verify
- grep → search pattern + verify count
- e2e → run Playwright/equivalent scenario

### Step 6: Produce report
Generate structured PASS/FAIL report with evidence for each check.

## Expected deliverable
- Validation report (PASS or FAIL with evidence)
- If FAIL: list of issues with file:line references

## Validation criteria (automated)
- [ ] All visual checks pass (UI projects)
- [ ] All code checks pass (no anti-patterns found)
- [ ] All runtime checks pass (endpoints/commands respond correctly)
- [ ] All acceptance tests pass

## Auto-validation flow
- **All pass** → proceed to Phase 4 (Test) automatically
- **Any fail** → return report to developer, developer fixes, re-validate (max 3 cycles)
- **3 failures** → escalate to human with full report and evidence
