# Phase 5.5: Security Audit (Auto-Validated)

## Responsible agent
`security`

## Objective
Automated security audit of the implementation. Verify OWASP Top 10 compliance, auth/authz, secrets detection, input validation, and injection prevention.

## Prerequisites
- Phase 5 (Review) completed
- All code changes committed

## Instructions

You are in **Phase 5.5 — Security Audit**. Your role is to find security vulnerabilities before deployment.

### Step 1: OWASP Top 10 scan
For each applicable category:
1. A01: Broken Access Control — verify route protection, authz checks
2. A02: Cryptographic Failures — check for weak algorithms, plaintext secrets
3. A03: Injection — SQL injection, XSS, command injection checks
4. A04: Insecure Design — verify security patterns in architecture
5. A05: Security Misconfiguration — check default configs, error messages
6. A06: Vulnerable Components — check dependencies for known CVEs
7. A07: Auth Failures — verify auth mechanisms, session management
8. A08: Data Integrity — verify data validation, serialization safety
9. A09: Logging Failures — verify security events are logged
10. A10: SSRF — verify URL validation, request filtering

### Step 2: Secrets detection
1. Grep for API keys, passwords, tokens in code
2. Check .env files are gitignored
3. Verify no secrets in commit history
4. Check no hardcoded credentials

### Step 3: Input validation
1. Verify all user inputs are validated (API endpoints, forms, CLI args)
2. Check for proper sanitization
3. Verify error messages don't leak internals

### Step 4: Auth/Authz verification
1. Verify all protected routes require authentication
2. Verify authorization checks on sensitive operations
3. Check token validation and expiry
4. Verify CORS configuration (web projects)

### Step 5: Produce report
Generate security audit report with risk levels.

## Expected deliverable
- Security audit report (PASS or FAIL with risk levels)

## Validation criteria (automated)
- [ ] No HIGH or CRITICAL vulnerabilities found
- [ ] No secrets in code or commit history
- [ ] All user inputs validated
- [ ] All protected routes require auth
- [ ] No known CVEs in dependencies

## Auto-validation flow
- **All pass (risk ≤ MEDIUM)** → proceed to Phase 6 (Deploy) automatically
- **HIGH/CRITICAL found** → return to developer for fix, re-audit (max 3 cycles)
- **3 failures** → escalate to human with full security report
